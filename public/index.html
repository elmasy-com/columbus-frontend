Columbus Project

Columbus Project is an API first subdomain discovery service.

A blazingly fast subdomain enumeration service with advanced features.

The goal of the project is to create an easy to integrate, yet powerful APIs.

More than 800M unique records and its growing!

Features

Speed

->

Result in seconds

Because every entry in MongoDB is indexed in the memory, Columbus can returns the result in 1 second on average. It returns the subdomains only to save CPU and RAM on the server side and save bandwith on both client- and server-side. The speed of a query is mostly depends on the client's bandwith.


The example below shows that Columbus can returns 739 subdomains for tesla.com in less than 0.5 second!

Lookup

->

Subdomain finder

Find subdomains for the given domain. Returns a list of all known subdomains for the given domain, which can be a JSON array or a newline delimetered string (for easy parsing). The domain parameter must be a valid domain (eg.: `example.com`).


The example below shows how easy to use the Lookup API.

Starts

->

Domain finder

Find domains that start with the given string. Returns a list of all known Second Level Domains thats start with domain, which can be a JSON array or a newline delimetered string (for easy parsing). The domain parameter must be a valid Second Level Domain (eg.: `example`) with at least five character.


The following example shows how this API endpoint works.

TLD

->

Top Level Domain finder

Find TLDs for the given domain. Returns a list of all known Top Level Domains for the given domain. The domain parameter must be only the Second Level Domain (eg.: example) without the TLD. Example: /api/tld/example returns ["com", "org", "net", ...].
